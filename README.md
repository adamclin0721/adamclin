# 個人網站

精簡的單頁個人網站，五個分頁：**首頁（心法）／投資儀表板／健康長壽／閱讀筆記／人生高光**。
純靜態、無框架、無外部 JavaScript 套件，部署在 GitHub Pages。

設計語彙是「精密儀器 / 製圖桌」——用一條量測刻度貫穿導覽列、儀表、項目軸與時間軸主幹，
寓意「同一把量尺，量市場，也量人生」。

---

## 檔案結構

```
你的-repo/
├── index.html        ← 渲染引擎，平常不用動
├── content.js        ← 所有文字 / 數字 / 文章都在這裡改
├── README.md         ← 本說明
├── reading/          ← 放「閱讀筆記」的 PDF（檔名建議用英數）
│   ├── outlive.pdf
│   ├── antifragile.pdf
│   └── …
└── images/           ← 放「人生高光」的圖片
    └── half-marathon.jpg
```

兩個核心檔（`index.html`、`content.js`）必須放在**同一層**。
`reading/` 與 `images/` 也放在同一層根目錄。

## 日常維護：只改 `content.js`

引擎與內容分離——**99% 的時候只要改 `content.js`，存檔後重新整理頁面就生效**，
不必碰 `index.html`。以下是各分頁的對應欄位。

### 首頁（座右銘 / 心法）

```js
home: {
  motto: "把投資與人生，都當成一套需要*紀律執行*的系統。",
  subtitle: "*低戲劇性。存活優先。長期主義。*\n波動會過去，流程會留下。",
  principles: [
    { tag: "SURVIVAL", title: "先求不敗，再求致勝。", note: "…" },
    // 想加一條心法就複製一整段 { } 貼上
  ],
}
```
`*星號*` 之間的字會變斜體強調；`\n` 是換行。

### 投資儀表板

平常**只改兩個地方**：最上面的 `lastUpdated`，以及每個指標的 `value`（最新數值）。

```js
dashboard: {
  lastUpdated: "2026-06-13",
  indicators: [
    { id:"tw_margin", label:"台股大盤融資維持率", short:"融資維持率",
      tag:"TWSE", value:158, thresholds:[140,150,160,170], unit:"%", decimals:0 },
    // …
  ],
}
```

- `thresholds` 是四條門檻，把情緒切成**五級**：
  `< 第1門檻＝極度恐懼`、`第1~2＝恐懼`、`第2~3＝中性`、`第3~4＝貪婪`、`≥ 第4＝極度貪婪`。
- 溫度軸的兩端（最小 / 最大）會由門檻**自動推導**，等距時門檻剛好落在 20 / 40 / 60 / 80%。
- `unit` 是單位（`"%"` 或留空 `""`），`decimals` 是小數位數。

### 健康長壽（長文）

卡片顯示「標題 + 摘要 + 閱讀全文 →」，有寫 `body` 的就能點進去讀全文。

```js
health: [
  {
    title: "文章標題",
    date: "2026·05", read: "8 分鐘", lead: "分類",
    desc: "卡片上的一句摘要。",
    tags: ["標籤A", "標籤B"],
    body: `這裡是長文。用反引號包起來，可以跨很多行。

一段一行，段落之間空一行。

## 這是大標
### 這是小標
- 這是項目
**粗體**、*斜體* 都可以。`,
  },
  // 新增文章＝複製一整段 { } 貼在最前面（最新的放最上面）
]
```
重點：短欄位用雙引號 `"…"`；**長文 `body` 用反引號 `` `…` ``**（這樣才能換行）。

### 閱讀筆記（PDF）

1. 把整理好的 PDF 丟進 `reading/` 資料夾。
2. 在這裡每本書填一段，`pdf` 指向那個檔名。

```js
reading: [
  { title:"Outlive", author:"Peter Attia", date:"2025 讀畢", stars:5,
    note:"把健康壽命當成主動工程。",
    tags:["長壽","預防醫學"],
    pdf:"reading/outlive.pdf" },
  // 40 幾本就照樣複製貼上
]
```
卡片顯示書名 + 一句話；點進去會**內嵌預覽整份 PDF**，並有「在新分頁開啟 / 下載」連結。
`stars` 是 1~5 顆星評分。

### 人生高光（時間軸）

**最上面＝最新**，往下變舊；要新增就加在陣列**最前面**。

```js
life: [
  { year:"2026", title:"最近的里程碑", desc:"說明…", now:true },
  { year:"2026", title:"另一個里程碑", desc:"…",
    image:"images/photo.jpg" },   // ← 想附圖才加這行
  // …
]
```
- `now: true` 會把節點標成**黃銅色**（代表「現在」），通常只留一個。
- 要附圖：把圖片放進 `images/` 資料夾，加一行 `image:"images/檔名.jpg"`；沒這行就不顯示圖。

## 編輯三原則（避免改壞）

1. 每一筆資料用大括號 `{ }` 包起來。
2. 欄位之間、每一筆之間都要有逗號 `,`。
3. 短文字用半形雙引號 `"…"`；長文章 `body` 用反引號 `` `…` ``。

> 小技巧：改完若頁面空白，多半是少了逗號或引號沒成對。
> 可在瀏覽器按 F12 看 Console 的錯誤訊息，或用 [jsonlint 之類的工具] 檢查括號是否成對。

## 部署 / 更新（GitHub Pages）

**第一次：**
1. 建立 repo（命名為 `你的帳號.github.io` 可得到乾淨網址）、設為 **Public**、勾 Add a README。
2. 把 `index.html`、`content.js`，以及 `reading/`、`images/` 資料夾上傳到**根目錄**。
3. Settings → Pages → Source 選「Deploy from a branch」→ `main` / `/(root)` → Save。
4. 等約一分鐘，Pages 區會出現網址；勾起 **Enforce HTTPS**。

**之後更新：** 直接在 GitHub 網頁點開 `content.js` → 鉛筆 ✏️ → 改字 → Commit，約一分鐘上線。
（新增 PDF / 圖片時，記得也把檔案上傳到 `reading/` 或 `images/`。）

五項指標都**手動維護**——沒有免費、可直接在瀏覽器抓取的公開 API，所以由你平常查的來源填入：

| 指標 | 平常查的地方 |
| --- | --- |
| 台股大盤融資維持率 | 證交所盤後資料 / 券商看盤軟體的「大盤融資維持率」 |
| 台股 >200 日均線比例 | 自成分股計算，或財經網站的市場廣度 |
| 美股 >200 日均線比例 | StockCharts `$SPXA200R`（或 MMTH 指標） |

| BofA 牛熊指標 | 美銀（BofA）每週報告，多由財經媒體轉述 |

## 小提醒

- **PDF 預覽：** 在自己電腦上用 `file://` 直接開檔，瀏覽器可能擋掉 PDF 內嵌（顯示空白，但「在新分頁開啟」仍可用）。**部署到 GitHub 後一切正常。**
- **字型：** 採 Google Fonts（Archivo / Spectral / IBM Plex Mono）；離線或載入失敗時會自動退回系統字型，版面不會壞。
- **無障礙：** 支援鍵盤左右鍵切換分頁、`Enter` 開啟文章、`prefers-reduced-motion`（關閉指針動畫），手機上導覽列可橫向捲動。
- **免責：** 投資儀表板僅為個人情緒觀察，非投資建議；健康長文為個人筆記，非醫療建議。
